执行语句查询超时配置

方法 1: 配置 conf/neo4j.conf 参数

  • 优点
    • 便于控制
    • 超时时会抛出异常，便于把控
  • 缺点
    • 偶发失效
    • 无法动态控制变更,需要重启服务.

方法 2: 配置 runTimeboxed

  • 优点
    • 精准控制
    • 必定生效
  • 缺点
    • 修改大量语句
    • 超时时返回数据为空
    • 正常返回时和没有 CALL 数据结构不同
